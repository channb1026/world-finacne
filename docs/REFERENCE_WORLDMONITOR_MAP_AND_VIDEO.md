# 参考：World Monitor 的地图与视频流

[World Monitor](https://github.com/koala73/worldmonitor) 在地图展示和视频流方面做得很好，本文整理其做法并与本项目的可改进点对照。

---

## 一、地图（Map）

### World Monitor 的做法

| 能力 | 说明 |
|------|------|
| **双引擎** | 3D 球体（globe.gl + Three.js）与 2D 平面（deck.gl + MapLibre）可运行时切换 |
| **45 个可开关图层** | 冲突、军事、基建、航班、船舶、电缆、管道、数据中心等，按需显示 |
| **8 个区域预设** | 全球 / 美洲 / 欧洲 / 中东 / 亚洲 / 非洲 / 大洋洲 / 拉美，一键视角 |
| **CII 热力图** | 国家不稳定指数以 choropleth 叠在地图上 |
| **URL 状态** | 中心、缩放、当前图层、时间范围编码进 URL，可分享/书签 |
| **瓦片与性能** | 合法瓦片范围控制、CDN 缓存、离线 PWA 支持（约 500 瓦片上限） |

### 本项目现状

- **Leaflet 2D**：单底图（Stadia Alidade Smooth Dark），`BoundedTileLayer` 已避免越界 404。
- **数据层**：仅国家/地区点（MapSpot），点击打开 RegionDrawer 看资讯。
- **无**：图层开关、区域预设、URL 同步、3D。

### 可借鉴的改进（按优先级）

1. **区域预设（低成本）**  
   在地图上方或侧边加一排按钮（全球 / 美洲 / 欧洲 / 亚太 / 中国等），点击后 `map.setView(中心, 缩放)`，用 Leaflet 现有 API 即可，无需换引擎。

2. **URL 同步视角（中成本）**  
   把 `center + zoom` 写入 `?lat= &lng= &zoom= `，页面加载与地图移动时读写 URL，便于分享当前视角。

3. **图层开关（中高成本）**  
   若以后有「交易所/央行/港口」等点层或面层，可做图层列表 + 勾选显示/隐藏，仍用 Leaflet 的 LayerGroup 即可。

4. **3D / 换引擎（高成本）**  
   仅当确实需要「球体 + 大量图层」时再考虑引入 globe.gl 或 deck.gl；当前以资讯点为主，Leaflet 足够。

## 二、视频流（Video Streams）

### World Monitor 的做法

| 能力 | 说明 |
|------|------|
| **30+ 直播源** | Bloomberg、Sky News、Al Jazeera 等，**HLS 原生播放**（非 iframe 嵌 YouTube） |
| **22 路热点摄像头** | 地缘热点地区直播画面，独立标签页（如 Iran/Attacks） |
| **idle-aware 播放** | 标签页不可见时暂停或降码，省带宽与电量 |
| **全屏模式** | 单路全屏观看 |
| **HLS** | 使用原生 `<video>` + MSE 或 hls.js 等，避免各站 iframe 策略限制 |

### 本项目现状

- **LivePanel**：仅提供「新窗口打开」链接（France 24、Al Jazeera、DW、CNBC、Bloomberg），不嵌入。
- **原因**：注释写明「嵌入易遇 Video unavailable」——多数新闻站禁止 iframe 嵌入，YouTube 嵌入也常受策略限制。

### 可借鉴的改进（按可行性）

1. **保留「新窗口打开」为主（推荐）**  
   与 WM 的「22 路 webcam 独立标签」思路一致：直播仍以跳转为主，避免跨域与策略问题，体验也稳定。

2. **HLS 直链嵌入（需有稳定源）**  
   若能找到**允许嵌入、且提供 HLS (.m3u8) 的财经直播源**，可在站内用 `<video>` + hls.js 嵌入一路「财经直播」窗口，并做：
   - 标签页不可见时 `video.pause()`（idle-aware）；
   - 提供全屏按钮。
   WM 的 30+ 直播是自有/授权 HLS 源，不依赖 YouTube iframe。

3. **直播入口与地图并列**  
   在布局上可参考 WM：直播区与地图同屏可见（如一侧面板或底部抽屉），点击某频道仍以新窗口打开，避免嵌入带来的不可用。

## 三、小结

| 方向 | World Monitor 亮点 | 本项目可落地动作 |
|------|---------------------|------------------|
| **地图** | 双引擎、多图层、区域预设、URL 状态 | 先做：区域预设 + 视角 URL 同步；有余力再加图层开关 |
| **视频** | HLS 原生、idle-aware、全屏 | 保持「新窗口打开」为主；若有 HLS 直链可单独做一路站内嵌入 + 暂停/全屏 |

地图上优先做「区域预设 + URL 同步」即可明显提升可用性和「像 WM 那样好用的感觉」；视频在无稳定 HLS 源前，维持当前跳转策略最稳妥。
